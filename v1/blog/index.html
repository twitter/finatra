Announcing Finatra Release 2.2.0!

by @capotej 30 Jun 2016

Finatra release 2.2.0 is now available from Maven Central.

This is our first release to support JDK 8 and Scala 2.11 only. We’ve also added a lot of useful scaladocs throughout the codebase, so take a look!

Please note that the published groupId for all Finatra artifacts has changed from com.twitter.finatra and com.twitter.inject to just com.twitter. You can find the published v2.2.0 artifacts here.

Here’s a list of some of the major changes, updates, and fixes:

  • The groupId of both finatra and finatra/inject published artifacts has changed from com.twitter.finatra and com.twitter.inject respectively to just com.twitter.
  • SLF4J api has been updated to 1.7.21 and examples and tests using Logback have been updated to 1.1.7. It is recommended that you also update your version if you are using the Logback logging implementation.
  • The Jackson Module Scala has been updated to version 2.6.5.
  • Support for any HTTP method in HTTP Controllers.
  • HTTP test utilities have been moved to a consistent package naming. Thus utilities formerly in com.twitter.finatra.http.test are now packaged com.twitter.finatra.http. As always these test utilities are available in the finatra-http test jar.
  • The utility method for running a “Warmup Handler” has been renamed from com.twitter.inject.server.TwitterServer#run to com.twitter.inject.server.TwitterServer#handle.
  • Fixed a bug in the handling of java callbacks which return a Future hidden as a java.lang.Object in the com.twitter.finatra.http.internal.marshalling.CallbackConverter.
  • Removal of deprecated packages and code.
  • Simplified and brought consistency to the Embedded testing utilities constructor arguments.
  • FilteredThriftClientModule and general thrift improvements to better capture and stat exceptions.
  • Narrowed visibility on classes and objects in internal packages which are not necessarily meant to be publicly exposed. It is expected that users depend only on the “public” framework APIs and not internal framework implementations.
  • Re-worked the server lifecycle to address confusion around naming and intent. Added scaladocs around the lifecycle to better aid in understanding of what the framework is providing on top of TwitterServer and addressed lot of testing utility issues including re-writing EmbeddedApp to make it simpler/easier to test cmd line applications (which is the central purpose of com.twitter.inject.app.App).

See the Finatra Github page or checkout our User Guide for more information on getting started.

Announcing the final Scala 2.10 and Java 7 release

by @capotej 26 Apr 2016

Our upcoming release: v2.1.6 will be our final release to support Scala 2.10 and Java 7.

Java 7 reached its end-of-life in April 2015 and we would like to begin using the Java 8 libraries in earnest. Once Scala 2.12 is released, we plan to add support for cross-compiling Scala 2.11 and 2.12. Until 2.12 is released we will only be supporting Scala 2.11.

New Testing System

Write tests for just a Controller instead of an entire app:

class SampleController extends Controller {
  get("/testing") {
    request => render.plain("hi").toFuture
  }
}



"GET /testing" should "be 200" in {
  val app = MockApp(new SampleController)
  val response = app.get("/testing")



  response.code should be(200)
  response.body should be("hi")
}

Finatra 1.5.1 Released

by @capotej 13 Jan 2014

Response internals fix

It is now possible to modify the response “out of band”, since Response is tied to Request via request.response. For example, in a filter:

class ExampleFilter
  extends SimpleFilter[Request, Response] {



  def apply(
    request: FinagleRequest,
    service: Service[Request, Response])
  ) = {
    request.response.headers.add(“X-Filtered”, “True”)
    service(request)
  }



}

See RequestResponseSpec for more details.

Finatra 1.5.0 Released

by @capotej 06 Jan 2014

This is the first release to be dual published for scala 2.9.x and 2.10.x, as such the artifactId has changed from finatra to finatra_2.9.2 or finatra_2.10, respectively.

Download

https://github.com/twitter/finatra/archive/1.5.0a.zip


Header API change

There is a breaking change in the way request.headers are accessed: it now returns an HttpHeaders object instead of the previous HeaderMap, which is now located at request.headerMap.

Changing all request.headers calls to request.headerMap should be sufficient. For more examples, see these commits: 6da615, 1dd624 and b95119.

Note: render.headers is not affected by this change.


SBT Support

The example app, Finatra itself, and the app generator are now all SBT compatible.
